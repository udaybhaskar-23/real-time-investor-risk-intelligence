# Model Documentation

## Overview

This document explains the machine learning model used in the Real-Time Investor Risk Intelligence Platform.

The project includes a baseline machine learning risk classification model that predicts whether current stock or portfolio conditions are classified as:

* Low Risk
* Medium Risk
* High Risk

The model is designed to support portfolio risk monitoring by using historical stock behavior, volatility signals, drawdown metrics, volume movement, and benchmark market indicators.

## Model Purpose

The purpose of the model is not to predict exact stock prices.

Instead, the model is designed to classify short-term market risk conditions based on recent historical patterns.

The model helps answer:

* Does the current portfolio show Low, Medium, or High Risk conditions?
* Which selected stocks show higher predicted risk?
* How confident is the model in its prediction?
* Which features are most important in the model?
* How does current market behavior compare with historical risk patterns?

## Model Type

The baseline model uses:

```text
Random Forest Classifier
```

A Random Forest model was selected because it is:

* Beginner-friendly
* Good for classification problems
* Able to handle non-linear patterns
* Useful for tabular financial features
* Easy to explain in a portfolio project
* Capable of producing feature importance values

The model training script is located at:


This script performs the following steps:

1. Downloads historical stock data

3. Creates benchmark features

6. Splits data into training and testing sets

## Target Variable

The model predicts:

```text
risk_label
```

The risk label is created using future 5-day return behavior.

## Label Creation Method

The target label is based on the next 5 trading days of return.

Formula:

```text
Future 5-Day Return = Close Price 5 Days Ahead / Current Close Price - 1
```

Label rules:

| Future 5-Day Return                            | Risk Label  |
| ---------------------------------------------- | ----------- |
| Less than or equal to -3%                      | High Risk   |
| Greater than -3% and less than or equal to +1% | Medium Risk |
| Greater than +1%                               | Low Risk    |

## Why Future 5-Day Return Is Used

Future 5-day return is used only to create the training label.

It is not used as a prediction feature.

This allows the model to learn from historical conditions that were followed by short-term downside, neutral, or positive outcomes.

## Train/Test Split

The dataset is split into:

```text
80% training data
20% testing data
```

The split uses stratification so that Low Risk, Medium Risk, and High Risk classes are represented in both training and testing sets.

## Model Configuration

The Random Forest model uses the following configuration:

```text
n_estimators = 300
max_depth = 8
min_samples_split = 10
min_samples_leaf = 5
class_weight = balanced
random_state = 42
```

## Class Weighting

The model uses:

```text
class_weight = balanced
```

This helps reduce the impact of class imbalance by giving more weight to underrepresented risk classes.

## Model Performance

The baseline model produced an accuracy of approximately:

```text
48.43%
```

This is acceptable for a first baseline version because financial market risk classification is difficult and noisy.

The model should be interpreted as a demonstration of machine learning workflow, not as a production-grade financial prediction model.

## Stock-Level Prediction

For each selected stock, the model predicts:

* Predicted risk level
* Confidence score
* Probability of Low Risk
* Probability of Medium Risk
* Probability of High Risk

## Portfolio-Level Prediction

The portfolio-level ML risk prediction is calculated by combining stock-level class probabilities using portfolio weights.

Formula:

```text
Portfolio Probability = Sum(Stock Risk Probability × Portfolio Weight)
```

The portfolio risk class with the highest weighted probability becomes the final portfolio ML risk level.

* Streamlit app integration

## Model Limitations

The model has several limitations:

* Accuracy is limited because financial markets are noisy
* The model does not predict exact stock prices
* The model does not include macroeconomic indicators yet
* The model does not use intraday data
* The model does not include earnings events
* The model does not include sector-level features
* The model may not generalize well during unusual market conditions
* Historical patterns do not guarantee future outcomes
* The model should not be used for real investment decisions
